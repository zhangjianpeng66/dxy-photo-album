## 一、项目概述

一个基于 Three.js 的 3D 卡片流相册网页应用。用户可以通过滚轮/拖拽浏览照片卡片，卡片可在多种 3D 阵型间切换（散落、横排、圆形、螺旋等），双击放大查看。背景为星空 + 品红色爱心双层粒子特效。

**部署方式**：纯静态站点，Vercel 免费部署，零成本。

---

## 二、技术栈

| 层 | 选型 | 理由 |
|---|------|------|
| 3D 渲染 | Three.js (CDN) | 一张 HTML 即可跑，零构建配置 |
| 开发预览 | Vite | 热更新，改代码即刷 |
| 内容配置 | config.json | 用户唯一需要编辑的文件 |
| 部署 | GitHub + Vercel | 免费，自动生成分享链接 |

## 三、卡片设计

### 3.1 卡片规格

- **背景卡片**：16 : 10 横版
- **照片**：9 : 16 竖版，不裁切不拉伸
- **等边距约束**：照片上边距 = 下边距 = 左边距 = 到中线的距离（统一为 M）
- **文案区右边距**：也等于 M，保证 `M | 照片 | M | 文案 | M` 均匀分布

### 3.2 文案排版（A 风文艺衬线）

```
第1行：English（衬线斜体小字，letter-spacing: 2px，color: #d090a0）
第2行：中 文（大字宽距，font-weight: 300，letter-spacing: 5px，color: #f5d5e0）
分隔 ：20px 品红细线（rgba(233,30,99,0.35)）
第4行：正文（小字左对齐，color: #b08090，letter-spacing: 1px）
```

### 3.3 文案内容格式（config.json）

```json
{
  "title_cn": "海边",
  "title_en": "Seaside",
  "text": "那天的风很温柔"
}
```

## 四、页面布局

### 4.1 标题动画

- **进场**（0s）：B 方案 — 中央大字 "dxy" + 副标题居中，电影片头感
- **过渡**（~1.5s）：字号缩小 + 位移到左上 + 透明度微降，ease-out 缓动
- **定格**（~2.5s+）：A 方案 — 左上角，占屏幕约 **1/20**，贴近边缘

### 4.2 标题内容

- 主标题：`dxy`
- 副标题：`forever lovers  jun zi hao qiu`

## 五、星空背景

- **密度**：中等
- **层次**：大/中/小三种尺寸，不同透明度，模拟真实夜空纵深
- **闪烁**：部分星星有微弱呼吸式闪烁

## 六、爱心粒子特效（双层混合）

### 6.1 背景层 — 大爱心缓慢飘落

- 尺寸：较大（~20-40px）
- 运动：从顶部缓慢飘落到底部，有水平漂移
- 速度：慢（类似樱花飘落）
- 数量：稀疏，画面中同时约 5-10 颗
- 透明度：较低，不抢眼

### 6.2 近景层 — 小爱心萤火流动

- 尺寸：较小（~8-16px）
- 运动：从下往上飘（萤火虫感），带曲线路径
- 速度：中等偏快
- 数量：较多，画面中同时约 15-25 颗
- 特效：带微光拖尾

### 6.3 粒子属性

- 颜色：品红系 (#e91e63, #f06292, #ec407a, #d81b60)
- 形状：爱心 ❤（Canvas 绘制或 SVG path）
- 3D 位置：分布在 Z 轴不同深度

## 七、3D 卡片交互

### 7.1 阵型（参考视频效果）

1. **散落** — 卡片随机分布，有轻微旋转
2. **横排** — 卡片水平排列成一行
3. **圆形** — 卡片围成圆环
4. **螺旋** — 卡片沿螺旋线排列
5. **十字** — 横竖交叉排列
6. **密集网格** — 卡片填满画面

### 7.2 交互方式

- **鼠标滚轮**：切换阵型或调整卡片间距
- **鼠标拖拽**：旋转整个卡片组
- **双击卡片**：放大查看单张照片详情

### 7.3 动画

- 阵型切换时使用 GSAP 或自定义缓动，卡片依次移动到目标位置
- 卡片有 stagger 延迟（每张卡片略微错开，形成波浪感）

## 八、文件结构

```

├── index.html          # 主页面
├── style.css           # 样式（标题动画等）
├── config.json         # 用户编辑的照片和文案配置
├── photos/             # 用户照片文件夹
│   ├── photo1.jpg
│   └── photo2.jpg
└── js/
    ├── main.js         # 主逻辑，初始化场景
    ├── stars.js        # 星空背景

    ├── cards.js        # 3D卡片系统 + 阵型
    └── interactions.js # 滚轮/拖拽/双击交互
```

## 十、部署流程

1. 创建 GitHub 仓库
2. 将项目文件推送到仓库
3. Vercel 关联 GitHub 仓库，自动部署
4. 生成公开链接 → 别人打开就能看
5. 更新内容：修改 config.json + 添加照片 → git push → Vercel 自动更新

---

## 十一、约束与边界

- 零成本：所有服务使用免费额度
- 纯静态：无后端、无数据库
- 文科生友好：唯一需要编辑的是 config.json
- 响应式：桌面端为主（3D 效果在移动端性能受限）
